# Prompt: Erkläre wie ein Senior Dev

> Lass dir Code so erklären, als würde ein erfahrener Kollege neben dir sitzen.

## Prompt

```
Erkläre mir diesen Code als wärst du ein Senior Developer der einen Junior onboardet.

Struktur:
1. **Was macht es?** — Ein Satz, kein Jargon
2. **Warum so?** — Welches Problem wird gelöst, welche Alternativen gab es
3. **Worauf aufpassen?** — Typische Fehler, Edge Cases, Gotchas
4. **Wie erweitern?** — Wenn ich Feature X hinzufügen will, wo anfangen?

Nutze Analogien aus dem Alltag wenn es hilft.
Keine trivialen Erklärungen ("hier wird eine Variable deklariert").
```

## Wann nutzen?

- Neues Projekt, unbekannte Codebase
- Komplexe Logik die du nicht sofort verstehst
- Onboarding neuer Teammitglieder
